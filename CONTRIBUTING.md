# Contributing to Water Balloon Launcher Simulation

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you agree to abide by the terms of our Code of Conduct. Please be respectful and inclusive in all interactions.

## Getting Started

### 1. Fork the Repository
Fork the repository to your own GitHub account and then clone it to your local machine.

### 2. Set Up the Development Environment
Follow the setup instructions in the [README.md](README.md) to set up both the backend and frontend.

### 3. Create a Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix-name
```

## Coding Standards

### Frontend
- Use **TypeScript** for all new code.
- Follow the existing component structure in `src/components`.
- Use clean, descriptive variable and function names.
- Run the linter before submitting: `npm run lint`.

### Backend
- Use **Python 3.10+**.
- Follow PEP 8 guidelines.
- Add docstrings to new functions and classes.

## Submitting a Pull Request

1. **Commit your changes**: Write clear, concise commit messages.
2. **Push to your fork**: `git push origin your-branch-name`.
3. **Open a Pull Request**: Describe your changes in detail and include screenshots if you've made UI improvements.

## Reporting Bugs

If you find a bug, please open an issue on GitHub with:
- A clear description of the bug.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Screenshots (if applicable).

Thank you for your contributions!
